📋 [개발 지시서] 안트그라비티 기반 날씨 브리핑 백그라운드 처리 시스템
======================================================================

1. 목적 및 개요
   - 목적: 유저별 지역 맞춤형 날씨 브리핑(오디오 소스)을 메인 스레드 병목 없이 백그라운드에서 실시간/주기적으로 생성 및 다운로드하고, 재생 시 음성 끊김 현상을 원천 차단함.
   - 핵심 과제:
     * Gemini API 및 Google Cloud TTS 호출 시 발생하는 429 에러(Too Many Requests) 방지.
     * 기기 내장(Native) TTS의 이질감을 극복하기 위한 고품질 오디오 파일 조립 및 크로스페이드 처리.

2. 안트그라비티(Ant-Gravity) 작업 큐(Queue) 구조
   - 오디오 생성 및 다운로드는 무겁고 네트워크 대기 시간이 길기 때문에, 안트그라비티의 비동기 워커(Worker) 패턴을 활용하여 단방향 파이프라인으로 처리합니다.

3. 세부 구현 스펙 및 기능 정의
   🛠️ 태스크 1: 429 에러 방지를 위한 레이트 리밋 및 백오프 설정
     - 지수 백오프(Exponential Backoff) 필수 적용:
       * 네트워크 불안정 또는 순간적인 429 에러 발생 시 즉시 재시도하지 않고 1s -> 2s -> 4s 형태로 대기 시간을 늘리며 최대 3회 재시도함.
     - 동시성 제한(Concurrency Limit):
       * 동시에 실행 가능한 TTS 변환 태스크의 풀(Pool) 개수를 1개로 제한하여 429 에러를 방지함.

   🎧 태스크 2: 오디오 음성 자연스러움 확보 (통문장 변환 및 조립)
     - 텍스트 빌딩:
       * "오늘 [제주시 노형동]의 오전 날씨는 [맑음]입니다. 현재 기온은 [섭씨 20도]로 안개주의보가 발효 중이니 출근길 운전에 유의하시기 바랍니다."
       * 위와 같이 변수가 포함된 문장 전체를 먼저 완성한 후 Google Cloud TTS(Neural2)로 변환함.
     - 오디오 크로스페이드(Crossfade) 구현:
       * '인트로 시그널 음악(효과음)' 뒤에 날씨 브리핑 음성을 이어서 출력할 때, 인트로 오디오가 끝나기 0.2초 전에 날씨 브리핑 오디오의 볼륨을 페이드인(Fade-in)하며 겹치도록 믹싱함.

   💾 태스크 3: 로컬 캐싱 및 유효기간(TTL) 관리
     - 데이터 유효기간: 생성된 날씨 브리핑 오디오 파일(.mp3)의 유효기간은 최대 45분~1시간으로 설정.
     - 캐시 체크 로직: 안트그라비티 큐에 작업이 들어오면, 로컬 스토리지에 해당 지역의 유효한 오디오 파일이 있는지 확인 후, 존재하면 API 호출을 스킵하고 해당 파일을 즉시 반환함.

4. 예외 처리 및 모니터링 (오프라인/인증 오류 대응)
   - TTS 글자 수 오버플로우 방지: 브리핑 프롬프트 단에서 토큰 및 글자 수 제한을 엄격히 걸어 구글 TTS 무료 한도를 과도하게 사용하지 않도록 방어함.
   - 태스크 실패 시 폴백(Fallback): 네트워크 오프라인 상태이거나 API key 미등록 등 구글 API 호출 실패 시, 기기 내장 TTS(expo-speech)로 매끄럽게 폴백하여 브리핑이 중단되지 않도록 조치함.

5. JSON 보안 및 Git 연동 가이드
   - OAuth 클라이언트 보안 JSON 파일(client_secret_...json)은 민감한 비밀키를 포함하고 있으므로 퍼블릭 GitHub 레포지토리에 절대 업로드되지 않아야 합니다.
   - 프로젝트 루트의 .gitignore에 해당 파일을 반드시 등록하여 커밋에서 제외하고, 구글 API Key는 사용자가 앱 내 설정 메뉴에서 보안 입력하도록 유도하여 안전하게 AsyncStorage에 캐시하여 관리합니다.
